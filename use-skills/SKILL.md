# use-skills — 智能技能调度器

输入你的需求，我自动为你调用最合适的一个或多个技能。你不需要记住哪个技能叫什么名字。

## 工作方式

### 步骤 1 — 理解需求

分析用户输入的自然语言，提取核心诉求。识别出：
- **主要意图** — 用户最想做什么
- **次要意图** — 是否有连带需求
- **目标对象** — 针对什么代码/项目

### 步骤 2 — 匹配技能

对照路由表找到匹配的技能。如果有多个匹配，按依赖关系排好顺序。

### 步骤 3 — 展示计划

向用户简要说明匹配结果，确认后再执行。格式：

```
→ 分析需求：<你理解的需求>
→ 匹配技能：<技能1> → <技能2>（如有）
→ 开始执行...
```

### 步骤 4 — 执行技能

读取匹配技能的 `SKILL.md`，严格按照其指令执行。效果等同于直接调用了该技能。

## 技能路由表

| 用户意图 | 触发词 | 匹配技能 | 技能职责 |
|---|---|---|---|
| 新项目初始化 | 新项目、初始化、搭建、脚手架、创建项目 | 0-启动 | 目录结构 → uv 环境 → pre-commit → git init |
| 方案规划 | 规划、方案、设计、PRD、拆任务、需求分析 | 1-规划 | 方案追问 → 术语表 → 接口设计 → PRD → 任务拆解 |
| 代码分析 | 分析、理解、概览、地图、看不懂、结构 | 2-分析 | 模块地图 + 数据流 + 关键入口 + 架构位置 |
| 快速原型 | 原型、验证、试错、不确定、快速验证、试试 | 3-原型 | 一次性代码快速验证设计决策 |
| 编码实现 | 开发、实现、编码、写代码、TDD、功能开发 | 4-开发 | TDD 红-绿-重构循环 |
| 代码审查 | 审查、检查、验收、review、质量、issue | 5-检查 | 双轴审查（标准+需求） / Bug报告 |
| 优化重构 | 优化、重构、改进、架构、代码混乱、耦合 | 6-优化 | 架构分析 / 重构计划 |
| 调试排查 | 调试、bug、排查、诊断、报错、崩溃、性能 | 7-调试 | 六阶段结构化调试 |
| 版本管理 | 版本、git、提交、推送、分支、回滚、保存 | 8-版本管理 | git 全流程 |
| 收尾整理 | 整理、交接、收尾、结束、下班、同步 | 9-最后整理 | 交接文档 + 知识同步 + 安全确认 |

## 组合规则

当需求涉及多个技能时，按下述顺序排列：

1. **分析先行** — 如果包含"分析"意图，先做 2-分析
2. **规划先行** — 如果包含"规划"意图，先做 1-规划
3. **原型先行** — 如果包含"验证"意图，在规划后、开发前做 3-原型
4. **开发/重构** — 中间执行
5. **检查收尾** — 最后做 5-检查、9-最后整理

示例：
- "分析一下这段代码，然后帮我重构" → 2-分析 → 6-优化
- "帮我验证一下这个数据模型，没问题就用 TDD 实现" → 3-原型 → 4-开发
- "帮我规划一个用户认证功能，然后实现它" → 1-规划 → 4-开发
- "看看这个项目的问题，做个优化方案" → 2-分析 → 6-优化

## 执行策略

### 单一技能
直接读取该技能的 `SKILL.md`，按照其全部指令执行。包括该技能内部的阶段流程、提问方式、输出格式。

### 多技能组合
按顺序依次执行每个技能。每个技能执行完毕后，简短汇总结果再进入下一个。全部完成后给出一份整体总结。

## 注意事项

- 不要同时执行两个技能，必须按顺序串联
- 每个技能执行时，严格遵循其 SKILL.md 中的规则（包括 `disable-model-invocation` 标记）
- 如果用户的需求无法匹配任何技能，直接说明并询问是否需要其他帮助
