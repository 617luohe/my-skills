# 自动迭代

多轮编排 skill，接管任务链按 PDCA 自动推进直到闭环或跳出。位于 skills 体系编排层，与 use-skills 平级。

## Language

**自动迭代**:
多轮编排 skill，接管任务链按 PDCA 自动推进直到闭环或跳出。
_Avoid_: loop、循环任务、自动化工作流

**任务链**:
有依赖顺序的可执行任务列表，每项是穿通一层的垂直切片（schema→逻辑→测试）。非静态——执行中发现新问题可动态追加。
_Avoid_: pipeline、检查清单、todo list

**任务项**:
任务链中的单个任务，对应一个独立子 Agent 执行。
_Avoid_: step、阶段（与五阶段冲突）

**任务状态**:
每个任务项的当前状态：`pending | in_progress | passed | failed | blocked`。`blocked` 表示任务无法自行完成，需人工决策。
_Avoid_: 进度、完成度

**门禁**:
任务完成后的二元判定节点（pass/fail），决定任务链推进方向。
_Avoid_: checkpoint、review、审批

**门禁报告**:
子 Agent 完成后的结构化产出：`{gate: "pass"|"fail", issues: [{severity, file, desc}], summary}`。
_Avoid_: 输出、结果、结论

**阻断项**:
阻止门禁通过的严重问题（severity=blocker），必须修复才能推进。
_Avoid_: 严重 bug、致命错误

**熔断**:
达到最大迭代轮数后的强制终止机制。针对总迭代轮数，非单个任务重试次数。
_Avoid_: limit、超时、强制退出

**迭代**:
PDCA 循环中的一轮：取任务→执行→门禁→决策推进。
_Avoid_: 循环轮次、round、cycle

**主 Agent**:
持有日志+任务链+门禁状态，负责决策引擎和下一轮调度。始终存活，上下文精简。
_Avoid_: 编排器、控制器、loop master

**子 Agent**:
每次迭代中 spawn 出来执行具体任务的独立 Agent（上下文隔离），只接收当前任务描述+上轮门禁结果+相关文档引用。
_Avoid_: worker、执行者、任务代理

**状态探查**:
五阶段之第 1 阶段：读取项目文档和代码形成当前状态基线，写入日志。
_Avoid_: 代码扫描、前置探查、context gathering

**目标澄清**:
五阶段之第 2 阶段：用户输入模糊时用 grill-me 决策树追问；输入清晰时跳过。
_Avoid_: 需求确认、需求澄清

**任务规划**:
五阶段之第 3 阶段：基于目标+状态基线拆解任务链，设定每项的门禁标准。优先评估用户路线合理性。
_Avoid_: 任务拆解、work breakdown

**迭代执行**:
五阶段之第 4 阶段：PDCA 循环体，主 Agent 调度子 Agent 逐个推进任务链。
_Avoid_: 主循环、执行阶段

**日志**:
单个 Markdown 文件（`LOOP_LOG.md`），三层结构：迭代摘要 + 决策追溯 + 门禁状态。人可读，且 loop 自身引用以调整方向。
_Avoid_: 记录、log、运行记录

**自动模式**:
用户授权执行后的默认模式，每轮决策引擎自动选择下一任务，无需重复确认；不能越过共享理解确认或初始执行授权。
_Avoid_: 全自动、无人值守

**执行授权**:
用户确认任务规划可执行并允许进入原型或开发。与共享理解确认分离，是迭代执行的前置门禁。
_Avoid_: 默认同意、自动开工

**手动模式**:
可选模式，每轮决策引擎推荐方案后等待用户确认再执行。启动时声明"需要确认"即启用。
_Avoid_: 审查模式、交互模式

**阻塞**:
任务状态之一。任务遇到无法自行解决的问题，需人工决策才能继续。不退出门禁循环，但暂停推进。
_Avoid_: 卡住、挂起

## Relationships

- **自动迭代**位于编排层，调用 **use-skills** 完成单次任务路由
- 一个**任务链**包含多个**任务项**，按依赖顺序排列
- 一个**迭代**执行一个**任务项**，产出一个**门禁报告**
- **主 Agent** 读取**日志** + **任务链** + **门禁报告**，决策下一轮
- **子 Agent** 执行任务项，产出门禁报告
- **门禁** pass → 任务状态变为 passed → 推进下一任务项
- **门禁** fail → 决策引擎判断：修复 / 追加新任务项 / 标记为 blocked
- 单任务连续 fail 不触发**熔断**；**总迭代轮数**达上限才熔断
- 模糊输入触发**目标澄清**（grill-me 决策树）；清晰输入也要形成目标摘要并获得共享理解确认
- **任务规划**完成后等待**执行授权**；获得授权后，**自动模式**才进入**迭代执行**

## Example dialogue

> **User:** "给项目加一个缓存层"
>
> **自动迭代（主 Agent）:** "收到。先探查项目状态……项目是 Python + Redis 可用。目标摘要如下，请确认共享理解。"
> "确认后拆解任务链：1) 缓存抽象层  2) 查询结果缓存装饰器  3) 写入失效策略  4) 缓存监控指标。是否授权按此任务链执行？"
>
> **[迭代 1]** "执行任务 1：缓存抽象层…门禁 pass ✅"
> **[迭代 2]** "执行任务 2：查询缓存装饰器…门禁 fail ❌，3 个阻断项。"
> "决策：任务 2 回退到 in_progress，追加修复子任务 2a。继续。"
> **[迭代 3]** "执行任务 2a：修复装饰器线程安全问题…门禁 pass ✅"
> **[迭代 4]** "执行任务 2：重新检查…门禁 pass ✅"
> …
> **[收尾]** "任务链全部通过。4 个任务项，5 轮迭代，0 个阻塞。日志已写入 LOOP_LOG.md。"
