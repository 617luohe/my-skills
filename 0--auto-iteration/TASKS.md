### Issue 5 — 迭代执行核心

在 SKILL.md 中定义：

- **阶段 4（迭代执行）**：PDCA 子循环
  - P（Plan）：取下一个 pending 任务项
  - D（Do）：主 Agent 调用 use-skills 路由到目标 skill，spawn 子 Agent
  - C（Check）：子 Agent 产出门禁报告，主 Agent 判定 pass/fail
  - A（Act）：pass → 标记完成推进；fail → 决策修复/追加/阻塞
- **决策引擎规则表**：门禁 fail 时的完整决策分支
- **主控/工人 spawn 协议**：子 Agent 接收的三要素（任务描述 + 上轮门禁结果 + 文件引用）
- **终止条件判断**：质量全部通过 / 人工接管 / 熔断（默认 15 轮）
- `--max-rounds=N` 覆盖熔断阈值

**验证**：决策引擎规则覆盖所有门禁结果组合；spawn 协议确保子 Agent 输入不超 3 项。

### Issue 6 — 手动模式 + 收尾

在 SKILL.md 中定义：

- **手动模式**：`--manual` 或自然语言"手动"触发
  - 每轮门禁判定后暂停：展示门禁报告 + 推荐下一步 + 等待确认（Y/n/改方向）

  - 汇总格式：<N> 个任务，<M> 轮迭代，<X> 个追加子任务，<Y> 个阻塞，耗时
- `--checkpoint=stage4` 每轮暂停

**验证**：手动模式每条暂停消息包含门禁报告+推荐；收尾汇总格式完整。

### Issue 7 — 自然语言推断 + 边缘情况

在 SKILL.md 中定义：

- **关键词信号映射表**：
  - "手动"/"先问我"/"让我确认" → mode=manual
  - "只规划"/"调研"/"只分析不写代码" → max-rounds=0
  - "不设上限"/"直到完美" → 熔断提到 25 轮
- **熔断处理**：达上限后标记剩余任务 blocked，写入日志，进入收尾
- **阻塞挂起**：blocked 任务暂停，推进下一个可用任务
- **空任务链**：探查后发现无需执行 → 直接收尾
- **子 Agent 执行异常**：spawn 失败 / 超时 → 重试 1 次，仍失败 → 标记 blocked
- **自动恢复**：检测 LOOP_LOG.md 存在且未完成 → 从最后 passed 任务之后继续

**验证**：每个边缘情况有对应处理逻辑。
