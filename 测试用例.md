## 验证清单

优化后，AI 应该：

### ✅ 判断能力
- [ ] 能识别简单任务（单文件、已定位、明确小改动）
- [ ] 能识别中等任务（新功能、方案不定、多模块）
- [ ] 能识别复杂任务（"自动帮我"、多轮迭代、门禁反馈）
- [ ] 能识别边界场景（模糊需求、陌生项目）
- [ ] 能识别外部智能体、跨模型和并行评审意图

### ✅ 路由能力
- [ ] 简单任务 → 直接路由到单个 skill
- [ ] 中等任务 → 规划 → 用户确认共享理解与执行授权 → 开发 → 检查 → 版本管理
- [ ] 复杂任务 → 移交 0--auto-iteration
- [ ] 多 Agent 协作 → 移交 0--Agent统筹
- [ ] 多轮闭环+多 Agent → 0--auto-iteration 主控并调用 0--Agent统筹
- [ ] 模糊需求 → 追问澄清后再路由

### ✅ 执行能力
- [ ] 每个 skill 执行完毕后简短汇总
- [ ] 串联时按顺序执行，不并行
- [ ] 最终给出整体总结
- [ ] 复杂任务移交时说明理由

### ✅ 用户体验
- [ ] 输出格式统一：`→ 任务规模 → 分析需求 → 匹配技能 → 开始执行`
- [ ] 判断理由清晰（"含'自动帮我'关键词 + 需要多轮验证"）
- [ ] 无需用户指定 skill 名称
- [ ] 路由表引用一致（CLAUDE.md 引用 use-skills）

## 回归测试

修改后需确保原有功能不受影响：

| 原有功能 | 测试方法 | 预期结果 |
|---------|---------|---------|
| 显式调用 skill | `/1-规划` | 直接进入该 skill |
| use-skills 单一匹配 | `/use-skills 分析代码` | 匹配 2-分析 |
| use-skills 多技能串联 | `/use-skills 规划并实现` | 1-规划 → 等待执行授权 → 4-开发 |
| 各 skill 的"完成后推进" | 4-开发完成 | 自动进 5-检查 |
| auto-iteration 调用 use-skills | 0--auto-iteration 内部任务 | 正确路由到目标 skill |

## 已知限制

1. **任务规模判断依赖关键词**：如果用户描述不含触发词，可能误判
2. **中等任务串联是固定的**：无法根据项目特性动态调整
3. **复杂任务的阈值主观**：需要实际使用反馈调整
