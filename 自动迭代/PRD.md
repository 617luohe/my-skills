# 自动迭代 — PRD

## Problem Statement

用户使用 skills 工作流体系时，一个任务往往需要**多轮迭代**才能闭环。例如：开发 → 审查 → 发现问题 → 修复 → 再审查。当前 use-skills 只能做**单次**智能路由，每次都需要用户手动发起下一轮。用户需要手动记住任务进度、决策历史、门禁状态——认知负担大，且容易被上下文膨胀打断节奏。

## Solution

**自动迭代**——编排层 skill，与 use-skills 平级。接收一个目标描述，自动完成：状态探查 → 目标澄清 → 任务拆解 → PDCA 迭代执行 → 收尾。质量门禁驱动，全自动推进直到闭环或上限熔断。

## User Stories

- As a **开发者**, I want **一句话启动完整开发流程**（探查→规划→执行→审查→闭环），so that **我不需要记住该用哪个 skill、不需要手动推进多轮迭代**
- As a **代码审查者**, I want **审查不通过时自动进入修复迭代**，so that **问题能自动修复而不需要我记住每一个 issue 然后手动调度**
- As a **项目负责人**, I want **完整体迭代记录（LOOP_LOG.md）可供溯源复盘**，so that **我能理解 AI 的决策路径和每轮的产出**
- As a **谨慎的开发者**, I want **可选的手动模式**，so that **高风险任务可以每轮审查后由我亲自确认才推进**
- As a **持续使用者**, I want **上下文隔离的子 Agent 机制**，so that **多轮迭代不会让 AI 上下文膨胀而降智**

## Implementation Decisions

### 架构定位
- 编排层 skill，与 `use-skills` 平级，目录 `my-skills/自动迭代/`
- 调用 `use-skills` 做单轮任务路由，不重复实现调度逻辑
- 调用 `1-规划` 的 grill-me 追问和任务拆解模式

### 五阶段流程
1. **状态探查** — 扫描项目代码/文档，写入基线
2. **目标澄清** — 目标模糊时 grill-me 追问（自动判断清晰度）
3. **任务规划** — 基于目标+基线拆解任务链（B+C 混合：自动拆解为主，用户路线先评估）
4. **迭代执行** — PDCA 循环：取任务→spawn 子 Agent→门禁→决策推进
5. **收尾** — 最终日志 + 任务完成汇总

### 主控+工人模式
- **主 Agent**：持有日志引用 + 任务链 + 门禁状态，决策引擎，始终存活
- **子 Agent**：每轮独立 spawn，只收当前任务 + 上轮门禁 + 文件引用，执行完即销毁
- 目的：防止上下文膨胀导致 AI 降智

### 质量门禁
- 结构化门禁报告：`{gate: "pass"|"fail", issues[], summary}`
- `pass` → 任务完成推进；`fail` → 决策引擎判断修复/追加/阻塞
- 统一门禁标准：有 blocker → fail

### 终止条件（三重）
- 质量通过（正常出口）
- 人工接管（用户中途介入）
- 上限熔断（默认 15 轮）

### 接口
- 零参数默认：`/自动迭代 <目标>`
- 自然语言关键词推断：手动、只规划、并行等
- 显式 flag：`--manual` `--max-rounds=N` `--skip-probe` `--task-chain=<file>` `--checkpoint=<stage>` `--context=<glob>`
- 自动恢复：检测 `LOOP_LOG.md` 状态为"未完成"时从中断点恢复

### 日志
- `LOOP_LOG.md` 单个文件，三层结构：迭代摘要 + 决策追溯 + 门禁状态
- 人可读，且主 Agent 引用以调整方向

## Testing Decisions

| 模块 | 测试方式 | 说明 |
|------|----------|------|
| 目标清晰度判断 | 用例集测试 | 给定多个目标描述，验证清晰/模糊判定准确性 |
| 任务链自动拆解 | 用例集测试 | 给定目标+项目基线，验证任务链完整性 |
| 门禁判定逻辑 | 用例集测试 | 给定门禁报告，验证 pass/fail 判定 |
| 决策引擎路径 | 用例集测试 | 给定门禁结果序列，验证路径选择（推进/修复/追加/阻塞） |
| 日志三层结构 | 格式验证 | 验证 LOOP_LOG.md 包含三层结构 |
| 手动模式暂停 | 人工验证 | 验证 --manual 模式下每轮暂停等待确认 |
| 上下文隔离 | 人工验证 | 对比直接循环调用 vs 主控+工人，观察上下文膨胀差异 |
| 端到端 | 人工验证 | 一个完整的小目标（如"加类型注解"）从头跑到尾 |

## Out of Scope

- **DAG 式并行任务执行** — 只支持线性依赖任务链。独立任务由用户分两次调用
- **跨会话状态恢复** — 不支持 `--resume`。LOOP_LOG.md 可辅助重跑但非真正恢复
- **差异化门禁标准** — 所有任务共享同一套 gate 判定
- **跨项目编排** — 仅在 `$PWD` 内运行
- **自定义日志格式和路径** — 固定 `LOOP_LOG.md`
- **子 Agent 模型独立选择** — 子 Agent 继承主 Agent 模型
