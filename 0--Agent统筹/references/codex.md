# 外援执行器参考：Codex CLI

## 使用前探测

首次调用前运行 `codex --version` 与 `codex --help`，确认当前版本的无头执行、模型、沙箱、会话恢复和结构化输出参数。不要把本文示例当成永久稳定接口。

安装和认证应以 Codex 官方文档为准。检测不到命令或尚未登录时，只报告缺口，不代替用户提交凭据。

## 分派能力

- 写任务使用工作区写入沙箱，默认禁止访问工作区之外的路径。
- 只读评审使用只读沙箱。
- 网络默认关闭；任务确需网络时单独授权并在最终报告中注明。
- 优先输出结构化事件流，并把最终消息写到宿主临时目录下的独立文件。
- 长提示词优先通过标准输入传入，避免命令行长度和转义问题。

## 会话

- 首次执行成功后从结构化事件中捕获会话标识，并原子写回本机注册表。
- 后续任务恢复同一会话，但同一会话不得被两个任务同时写入。
- 恢复时若模型不兼容，保留客户端认可的模型或使用默认模型，并更新注册表。

## 验证与异常

- 记录退出码、最终消息路径和结构化输出路径。
- 子进程控制、浏览器沙箱或杀进程操作被系统拒绝时，标记对应验证未完成，不反复尝试扩大权限。
- 无法列出模型时使用客户端默认模型，不猜测模型名称。
